『ElasticSearch』- Query DSL(Domain Specific Language)

发表于 2018-05-29 | 分类于 ElasticSearch |

ElasticSearch提供了基于json的完整查询DSL来定义查询。查询DSL包括两种子句叶查询子句和复合查询子句。查询子句的行为有所不同，具体取决于它们是在 查询上下文(Query Context)还是过滤器的上下文(Filter Context)中使用。

叶查询子句：

叶查询子句(Leaf query clauses)中寻找一个特定的值在某一特定领域，如 match，term或 range查询。这些查询可以自己使用。

复合查询子句：

复合查询子句(Compound query clauses)包装其他叶或复合查询，并用于以逻辑方式（如bool或dis_max查询）组合多个查询 ，或者改变它们的行为（如 constant_score查询）。

Query Context：

在查询过程中穿了判断文档是否满足查询条件之外，ES还会计算一个_score来标识查询结果的匹配程度，旨在判断目标文档和查询条件匹配的有多好(吻合程度)。
